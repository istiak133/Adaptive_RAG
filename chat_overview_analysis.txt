1. WHAT IS THE SYSTEM?
----------------------
You need to build an Adaptive Document Preparation System (a CLI-based smart exam prep application). 
The application must:
- Read a multi-section PDF document (~50 pages, 10 sections) - SLATEFALL_DOSSIER.pdf.
- Allow the user to select one or more sections to study (e.g., Section 5 and Section 8).
- Generate a set of Multiple Choice Questions (MCQs) derived from those sections using an LLM (default 5 MCQs per section).
- Collect the user's answers and score them, providing correct answers and detailed clarifications for every wrong answer.
- Persist a Knowledge Base (KB) of each preparation session: what was asked, what was answered correctly, and what was answered incorrectly.
- On subsequent runs over the same sections, adapt the question set by focusing on the user's historical weak areas (topics with consistent wrong answers) while avoiding excessive repetition of questions already mastered.

2. CORE WORKFLOW (PREP FLOW)
----------------------------
INPUT  : User supplies section IDs to study (e.g., sections 5 and 8).
STEP 1 : System checks the KB for any prior prep history for those sections.
STEP 2 : Generate N MCQs using an LLM (N is configurable, default = 5 per section).
STEP 3 : Present questions to the user and collect answers.
STEP 4 : Score the session.
         - For each correct answer: mark as correct.
         - For each wrong answer: show correct answer + concise explanation.
STEP 5 : Update the Knowledge Base with the performance data.
STEP 6 : Save a human-readable snapshot of the KB state after the session (kb_snapshot_iterX.json).

3. SCENARIOS TO SUPPORT
-----------------------
- Scenario A (Cold Start): A user prepares for sections they have never studied before. The system generates questions covering the selected sections evenly.
- Scenario B (History-Adaptive): A user prepares for sections they have studied before.
  * Iteration 1: User studies Sections 5 and 8 (Cold Start). User gets some questions wrong on specific topics (e.g., Combat Doctrine, Echo Lock). KB stores this.
  * Iteration 2: User studies Sections 6, 8, and 9. KB shows they previously struggled with Section 8 topics. The system prioritizes those weak areas in Section 8 questions. Sections 6 and 9 are new (Cold Start).
  * Iteration 3: User studies Section 8 again. KB shows history from Iteration 1 and 2. System tracks progress. Topics got wrong twice are high priority; topics got wrong once but corrected are low priority (mastered).

4. DOSSIER STRUCTURE (SLATEFALL_DOSSIER.pdf)
---------------------------------------------
The dossier has 10 sections detailing the metahuman operative SLATEFALL (Inez Calvache-Renström):
- Section 1: Identity and Public Profile (biographical data, history, spousal access).
- Section 2: Powers, Abilities, and Documented Limits (Inertial Suspension core mechanic, Drift Read precognitive sense, Hold, cooldown matrix, contact exclusion).
- Section 3: Professional History and Chronology (military career, Cerro Castillo awakening incident, promotion timeline).
- Section 4: Operational Gear and Equipment (neural telemetry wrist units, Reset Bracers, Yatiri-3 burst radio, Valparaíso armory munitions).
- Section 5: Operational Tactics and Combat Doctrine (Sequential Suspension DSS, Three-Two-One Rule, combat directives, departures like Salta/Mendoza).
- Section 6: Allies, Networks, and Known Affiliations (PAMC organization, Cell Halcón tracker/medic).
- Section 7: Threat Assessment and Adversaries (Tier 1-4 definitions, Quebrantadero, Tropopausa pressure manipulator).
- Section 8: Safehouses, Facilities, and Operational Bases (Cuartel Valparaíso, network nodes).
- Section 9: Declassified Engagement Case Files (Antofagasta containment, Mendoza engagement, Salta inquiry).
- Section 10: Reference Index and Glossary (Reference tables, codename mappings, document revision history).

5. KEY EVALUATION DIMENSIONS
-----------------------------
- Functional Correctness (30%): Prep flow must execute properly for both scenarios.
- Knowledge Base Design (20%): Schema must be well-thought-out, queryable, and relationally sound.
- Retrieval / Adaptation (20%): Iteration 2+ must genuinely use history to shape new questions.
- Code Quality (10%): Readability, structure, modularity, testing, logging, error handling.
- Documentation (15%): README clarity, setup instructions, architecture documentation.
- Optional Enhancements (5%): Spaced repetition, regression detection, quality scoring.

3. What You Build (কী তৈরি করতে হবে)
3.1 Core System — Prep Flow (মূল কাজের ধাপ)

PREP FLOW (পুরো কর্মপ্রবাহ):
INPUT  → ইউজার section ID দেবে (যেমন: section 3, 7)
STEP 1 → সিস্টেম KB চেক করবে — এই section-এ আগে কোনো prep হয়েছে কি না?
STEP 2 → LLM দিয়ে N টা MCQ জেনারেট করবে (N = configurable, default = প্রতি section এ ৫টা)
STEP 3 → প্রশ্ন দেখাবে, ইউজারের উত্তর নেবে
STEP 4 → স্কোরিং করবে:
          ✅ সঠিক উত্তর → "correct" মার্ক
          ❌ ভুল উত্তর → সঠিক উত্তর + সংক্ষিপ্ত ব্যাখ্যা দেখাবে
STEP 5 → ফলাফল KB-তে সেভ করবে (প্রশ্ন, উত্তর, স্কোর ইত্যাদি)
3.2 Knowledge Base (KB) Design — ডেটাবেস ডিজাইন
KB-কে এই query গুলো সাপোর্ট করতে হবে:

Query Pattern	ব্যাখ্যা
Section-wise history	কিছু section ID দিলে, ওই section-এর আগের সব prep session দেখাবে
Question-level results	কোনো session-এ কোন প্রশ্ন ঠিক/ভুল হয়েছিল
Weak-area identification	কোন topic/প্রশ্নে বারবার ভুল হচ্ছে (adaptive weighting-এর জন্য)
KB Snapshot	যেকোনো session শেষে KB-র অবস্থা export করতে পারবে

KB Snapshot = সবচেয়ে সাম্প্রতিক ৫টা session-এর human-readable export। Reviewer এটা দেখে যাচাই করবে যে history ঠিকমতো store হচ্ছে কি না এবং adaptive prompting আসল ডেটার উপর ভিত্তি করছে কি না।

Storage Backend: তুমি যেকোনো কিছু ব্যবহার করতে পারো — SQLite, PostgreSQL, vector store, plain JSON, বা hybrid। তবে schema এবং access pattern README-তে document করতে হবে।

class TokenBudget:
    def __init__(self, model_context_limit=8000):
        self.total_limit = model_context_limit
        self.output_reserve = 2500      # LLM এর response এর জন্য রাখো
        self.available = self.total_limit - self.output_reserve  # 5500 বাকি
        
        # বাকি 5500 ভাগ করো:
        self.system_prompt_budget = 400       # fixed
        self.instructions_budget = 300        # fixed

        self.content_budget = 3600            # বাকিটা PDF chunks এর জন্য

৩. Smart Chunk Selection — সব chunk পাঠাবে না, budget-এর মধ্যে best গুলো বাছাই
python


def select_chunks_within_budget(relevant_chunks, token_budget=3600):
    """
    ChromaDB থেকে relevant chunks পেয়েছো, কিন্তু সব পাঠালে 
    budget exceed হবে। তাই priority অনুযায়ী select করো।
    """
    selected = []
    used_tokens = 0
    
    # chunks আগে থেকেই relevance score দিয়ে sorted
    for chunk in relevant_chunks:
        chunk_tokens = count_tokens(chunk['text'])
        
        if used_tokens + chunk_tokens <= token_budget:
            selected.append(chunk)
            used_tokens += chunk_tokens
        else:
            break  # budget শেষ, আর নেওয়া যাবে না
    
    return selected, used_tokens
মানে: ChromaDB থেকে হয়তো ১০টা relevant chunk পেলে, কিন্তু budget-এ ৫টা ধরবে। তাই সবচেয়ে relevant ৫টা নাও, বাকি বাদ দাও।

# BAD — crash হবে:
questions = json.loads(llm_response)
# GOOD — resilient:
def parse_llm_json(response_text):
    # Step 1: markdown code block strip
    text = response_text.strip()
    if text.startswith("```"):
        text = text.split("```")[1]
        if text.startswith("json"):
            text = text[4:]
    
    # Step 2: JSON খোঁজো text এর মধ্যে
    start = text.find('[')  # or '{'
    end = text.rfind(']') + 1  # or '}'
    if start != -1 and end > start:
        text = text[start:end]
    
    # Step 3: parse try করো
    try:
        return json.loads(text)
    except json.JSONDecodeError:
        # Step 4: fail হলে retry with different prompt
        return None

def validate_single_mcq(q):
    errors = []
    
    # Check: 4 unique choices আছে কি না
    choices = [q['choice_a'], q['choice_b'], q['choice_c'], q['choice_d']]
    if len(set(choices)) < 4:
        errors.append("Duplicate choices found")
    
    # Check: correct answer A/B/C/D এর মধ্যে কি না
    if q['correct_answer'] not in ['A', 'B', 'C', 'D']:
        errors.append(f"Invalid correct answer: {q['correct_answer']}")
    
    # Check: question text খালি না
    if not q.get('question_text', '').strip():
        errors.append("Empty question text")
    
    # Check: explanation আছে কি না
    if not q.get('explanation', '').strip():
        errors.append("Missing explanation")
    
    return errors

def verify_source(question, source_quote, chunk_text):
    # Fuzzy match — exact match না পেলেও 80%+ similar হলে OK
    from difflib import SequenceMatcher
    ratio = SequenceMatcher(None, source_quote.lower(), chunk_text.lower()).ratio()
    if ratio < 0.5:
        return False, "Source quote not found in chunk — possible hallucination"
    return True, "Verified"

def generate_mcqs_with_retry(section_chunks, count=5, max_retries=3):
    for attempt in range(max_retries):
        try:
            questions = call_llm(section_chunks, count)
            validated = [q for q in questions if validate_single_mcq(q)]
            if len(validated) >= count:
                return validated[:count]
            # কম পেলে, বাকিগুলোর জন্য retry
            count = count - len(validated)
        except APITimeoutError:
            wait_time = 2 ** attempt  # exponential backoff: 1s, 2s, 4s
            time.sleep(wait_time)
            continue
        except RateLimitError:
            time.sleep(60)  # rate limit হলে ১ মিনিট অপেক্ষা
            continue
    
    # সব retry fail হলে, graceful degradation
    raise MCQGenerationError(
        f"Failed to generate MCQs after {max_retries} retries. "
        f"Generated {len(validated)} of {count} questions."
    )

def determine_difficulty(section_id, topic_performances):
    avg_score = sum(t['times_correct'] / max(t['times_asked'], 1) 
                    for t in topic_performances) / len(topic_performances)
    
    if avg_score >= 0.85:
        return "hard"
    elif avg_score >= 0.50:
        return "medium"
    else:
        return "easy"
5. Question Quality Scoring
প্রতিটা generated MCQ-র quality score বের করো:

def score_question_quality(question, chunk_text):
    score = 0
    reasons = []
    
    # 1. Source verification (hallucination check)
    if question.source_context and question.source_context in chunk_text:
        score += 30
    else:
        reasons.append("Source not verified")
    
    # 2. All choices unique
    choices = [question.choice_a, question.choice_b, question.choice_c, question.choice_d]
    if len(set(choices)) == 4:
        score += 20
    else:
        reasons.append("Duplicate choices")
    
    # 3. Explanation exists and is substantial
    if len(question.explanation) > 50:
        score += 20
    else:
        reasons.append("Weak explanation")
    
    # 4. Question length reasonable (not too short, not too long)
    if 20 < len(question.question_text) < 500:
        score += 15
    else:
        reasons.append("Question length unusual")
    
    # 5. Distractors are plausible (choices are not obviously wrong)
    if all(len(c) > 5 for c in choices):
        score += 15
    else:
        reasons.append("Some choices too short/implausible")
    
    return score, reasons
    
# Score < 50 → reject, regenerate
# Score 50-80 → accept with warning
# Score > 80 → good quality

def export_kb_snapshot(session_id):

            JOIN answers a ON q.id = a.question_id
            WHERE q.session_id = ?
        """, (session['id'],)).fetchall()

                    "correct_answer": q['correct_answer'],
                    "is_correct": bool(q['is_correct'])

            ],
            "weak_topics": [q['topic'] for q in questions if not q['is_correct']],

        }
        snapshot["recent_sessions"].append(session_data)

    snapshot["adaptive_state"] = {

            }
            for row in db.execute("SELECT * FROM topic_performance").fetchall()

    }
    
    return snapshot

def simulate_answers(session, correct_ratio=0.65):
    """
    Realistic simulation:
    - Weak topics (high weight) → বেশি ভুল হওয়ার সম্ভাবনা
    - Strong topics (low weight) → বেশি ঠিক হওয়ার সম্ভাবনা
    - Not purely random — pattern shows learning
    """
    for question in session.questions:
        topic_weight = get_topic_weight(question.section_id, question.topic)
        
        # weight বেশি → ভুল করার chance বেশি
        if topic_weight > 2.0:
            correct_prob = 0.3   # weak topic → ৩০% chance ঠিক
        elif topic_weight > 1.0:
            correct_prob = 0.6   # moderate → ৬০% chance
        else:
            correct_prob = 0.85  # strong → ৮৫% chance
        
        is_correct = random.random() < correct_prob
        
        if is_correct:
            user_answer = question.correct_answer
        else:
            wrong_choices = ['A','B','C','D']
            wrong_choices.remove(question.correct_answer)
            user_answer = random.choice(wrong_choices)
        
        save_answer(question.id, user_answer, is_correct)
এটা important কারণ — Reviewer outputs দেখে বুঝবে তোমার adaptive system কাজ করছে কি না। যদি fully random answer দাও, তাহলে pattern inconsistent হবে এবং adaptive behavior demonstrate করা কঠিন হবে।

def update_mastery_after_answer(question_id, is_correct):
    """
    একটা উত্তরের পরে সব affected topic-এর mastery update করো
    """
    
    # Step 1: এই question কোন কোন topic test করেছিল?
    question_topics = db.execute("""
        SELECT qt.topic_id, qt.is_primary, q.section_id
        FROM question_topics qt
        JOIN questions q ON qt.question_id = q.id
        WHERE qt.question_id = ?
    """, (question_id,)).fetchall()
    
    for qt in question_topics:
        topic_id = qt['topic_id']
        section_id = qt['section_id']
        is_primary_topic = qt['is_primary']
        
        # Impact factor: primary topic = full impact, secondary = partial
        impact = 1.0 if is_primary_topic else 0.5
        
        # ──── Global topic_mastery update ────
        mastery = get_or_create_topic_mastery(topic_id)
        
        mastery['times_asked'] += 1
        if is_correct:
            mastery['times_correct'] += 1
            # streak: ভুলের পরে ঠিক → reset to +1, ঠিকের পরে ঠিক → +1
            mastery['current_streak'] = max(mastery['current_streak'], 0) + 1
        else:
            mastery['times_wrong'] += 1
            mastery['current_streak'] = min(mastery['current_streak'], 0) - 1
        
        # Weight recalculate
        mastery['weight'] = calculate_weight(
            times_wrong=mastery['times_wrong'],
            times_correct=mastery['times_correct'],
            streak=mastery['current_streak'],
            impact=impact
        )
        
        # Mastery check: পরপর ৩ বার ঠিক = mastered
        if mastery['current_streak'] >= 3:
            mastery['is_mastered'] = True
        
        # Regression check: mastered ছিল কিন্তু ভুল করলো
        if mastery['is_mastered'] and not is_correct:
            mastery['is_mastered'] = False
            mastery['weight'] = max(mastery['weight'], 2.5)  # re-prioritize
        
        save_topic_mastery(mastery)
        
        # ──── Section-specific mastery update ────
        sec_mastery = get_or_create_section_topic_mastery(section_id, topic_id)
        
        sec_mastery['times_asked'] += 1
        if is_correct:
            sec_mastery['times_correct'] += 1
        else:
            sec_mastery['times_wrong'] += 1
        
        sec_mastery['weight'] = calculate_weight(
            times_wrong=sec_mastery['times_wrong'],
            times_correct=sec_mastery['times_correct'],
            streak=mastery['current_streak'],  # global streak use করো
            impact=impact
        )
        
        save_section_topic_mastery(sec_mastery)
def calculate_weight(times_wrong, times_correct, streak, impact=1.0):
    """
    Weight formula:
    - Base = 1.0 + (wrong * 0.5 * impact)
    - Streak penalty/bonus
    - Mastery decay
    """
    base = 1.0 + (times_wrong * 0.5 * impact)
    
    if streak < 0:  # consecutive wrong
        base *= (1.0 + abs(streak) * 0.3)
    elif streak >= 2:  # consecutive correct
        base *= 0.3
    elif streak >= 3:  # mastered
        base = 0.1
    
    return round(min(base, 5.0), 2)  # max 5.0 cap

def allocate_questions(section_id, total_questions=5):
    """
    Weight অনুযায়ী কোন topic থেকে কয়টা question আসবে সেটা বের করো
    """
    
    # Section-এর সব topic আর তাদের weight নাও
    topics = db.execute("""
        SELECT t.id, t.name, st.depth, st.relevance_score,
               COALESCE(tm.weight, 1.0) as global_weight,
               COALESCE(stm.weight, 1.0) as section_weight,
               COALESCE(tm.is_mastered, FALSE) as is_mastered
        FROM section_topics st
        JOIN topics t ON st.topic_id = t.id
        LEFT JOIN topic_mastery tm ON t.id = tm.topic_id
        LEFT JOIN section_topic_mastery stm 
            ON stm.section_id = st.section_id AND stm.topic_id = t.id
        WHERE st.section_id = ?
    """, (section_id,)).fetchall()
    
    # Effective weight = global × section × depth_multiplier
    for t in topics:
        depth_mult = {'primary': 1.0, 'secondary': 0.6, 'mention': 0.3}[t['depth']]
        
        if t['is_mastered']:
            t['effective_weight'] = 0.1  # mastered = almost skip
        else:
            t['effective_weight'] = t['global_weight'] * depth_mult
    
    # Weight normalize করে question allocate করো
    total_weight = sum(t['effective_weight'] for t in topics)
    
    allocation = {}
    remaining = total_questions
    
    # Sort by effective_weight descending
    topics_sorted = sorted(topics, key=lambda t: t['effective_weight'], reverse=True)
    
    for t in topics_sorted:
        if remaining <= 0:
            break
        
        # Proportional allocation
        share = (t['effective_weight'] / total_weight) * total_questions
        count = max(0, min(remaining, round(share)))
        
        # Minimum 1 if weight > 2.0 (weak topic must get at least 1)
        if t['effective_weight'] > 2.0 and count == 0 and remaining > 0:
            count = 1
        
        allocation[t['name']] = count
        remaining -= count
    
    # বাকি questions new/exploration topics এ দাও
    if remaining > 0:
        unasked = [t for t in topics_sorted if t['name'] not in allocation]
        for t in unasked[:remaining]:
            allocation[t['name']] = 1
    
    return allocation
